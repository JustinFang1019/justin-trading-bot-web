## 先說結論

我剛生成的圖比較像「新版產品設計總覽」，不是完全照原案 UI 重畫。

原案實際 UI 是 LINE Bot 內的 Flex Message 系統，功能比我圖上呈現的更多，也更貼近日常使用：

- 掃描小卡
- 個股查詢卡
- 策略卡
- 權證卡
- 回測卡
- RS 排名
- 績效追蹤
- 教學選單
- Rich Menu
- 管理員指令與群組權限

我的概念圖則偏向「把資訊層級重新整理」：

- 讓重點數字更快被看到。
- 把策略理由、風報比、停損、RS、KD 收斂集中呈現。
- 把管理監控做成獨立 dashboard 感。
- 視覺更乾淨、少一點深色大區塊與 emoji 依賴。

所以最好的方向不是二選一，而是用原案功能當骨架，用新版視覺邏輯做整理。

## 原案 UI 的實際特性

### 優點

- 已經非常 LINE-native，Flex Message 結構完整。
- 功能完整，從選股、查詢、權證、回測到教學都有。
- 對使用者很實用，按鈕可以直接導 Yahoo K 線、權證、財報、回測。
- 掃描結果有分數、連續天數、KD、壓力、進場區、停損、風報比。
- RS 排名是 compact list，適合 LINE bubble 的高度限制。
- 績效追蹤已按日期分組，且有資料新鮮度警示。
- Rich Menu 已經有主入口：大盤、掃描、RS排名、自選、進階、說明。

### 可以改善

- 各張卡片的視覺語言不一定完全一致。
- 深色 header 很醒目，但多張卡連續看時會有壓迫感。
- emoji 承擔太多資訊分類，專業感會略受影響。
- 掃描小卡資訊很多，第一眼不一定知道「現在該不該看這檔」。
- 權證、回測、策略、績效的資訊密度很高，適合老使用者，新使用者可能要學一下。
- 管理員監控資訊分散在指令與推播中，尚未形成一張清楚的「系統狀態卡」。

## 新版概念圖的特性

### 優點

- 資訊層級比較清楚：先看訊號，再看價格，再看理由。
- 狀態色比較克制：紅綠黃藍各有用途。
- 更像一套產品設計系統，可延伸到多張卡。
- 把「管理監控」也設計成正式 UI，而不是只靠文字 log。
- 適合做成下一版 LINE Flex 的共同設計語言。

### 不足

- 它不是照原案功能完整還原。
- 目前沒有呈現教學卡、回測卡、績效追蹤、Rich Menu 的細節。
- 有些畫面比較像 dashboard，但 LINE Flex 實際上會受 bubble 寬度與 payload 大小限制。
- 它用到小圖表與 sparklines，若要在 LINE Flex 中實作，需要用圖片、URI 或簡化成文字/條狀資訊。

## 融合版設計原則

### 1. 保留原案功能，不推倒重來

原案最有價值的是功能已經跑過很多情境：

- 掃描。
- 查詢。
- 權證。
- 回測。
- 教學。
- 績效。
- 權限。
- quota 與 cache。

融合版應該先整理 UI，而不是重寫所有功能。

### 2. 統一卡片語言

每張卡都使用相同的結構：

```text
Header：股票 / 模組名稱 + 狀態 badge
Summary：最重要的 1 到 3 個數字
Reason：為什麼出現這個訊號
Action：下一步按鈕
Meta：資料日期、快取、quota、來源
```

### 3. 掃描小卡要更像決策摘要

原案欄位可以保留，但順序建議改成：

1. 代號、名稱、訊號等級、分數、RS。
2. 現價與盤中/收盤狀態。
3. 進場區、停損、壓力、風報比。
4. KD / BOX / MA / 連續天數。
5. 三個主要按鈕：策略、權證、回測。

### 4. 權證卡要從「資料很多」變成「篩選理由清楚」

建議每檔權證顯示：

- 權證代號與名稱。
- 價格。
- Delta。
- 實質槓桿。
- 到期天數。
- 價內外程度。
- 流動性狀態。
- 風險標籤。

並分成：

- 推薦。
- 可觀察。
- 不建議。

### 5. 新增 Admin 狀態卡

原案已有很多營運保護，但 UI 可以更集中：

- 今日掃描時間。
- 掃描漏斗數量。
- K 棒 cache 新鮮度。
- FinMind quota。
- LINE quota。
- 權證 cache。
- Google Sheets 寫入狀態。

這張卡很適合管理員每天看一次。

## 建議融合版 UI 清單

第一階段可以先做 6 張核心卡：

1. 掃描結果卡：每日推播主體。
2. 個股摘要卡：使用者打股票代號時回覆。
3. 策略詳情卡：從掃描卡點「策略」。
4. 權證推薦卡：從個股或掃描卡點「權證」。
5. RS 排名卡：保留 compact list，但視覺更新。
6. Admin 狀態卡：管理員用。

第二階段再做：

1. 回測卡。
2. 績效追蹤卡。
3. 教學選單。
4. Rich Menu 重新設計。
