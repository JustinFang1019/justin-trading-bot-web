# Mobile Redesign Proposal

Last updated: 2026-05-06 Asia/Taipei

## Goal

Turn `justin-trading-bot-web` into a mobile-first companion viewer for the fuller `JustinFang1019/justin-trading-bot` LINE tool.

The fuller LINE tool remains the source of truth for:

- Strategy logic in `stock_scanner/patterns.py`
- Flex card content and builders in `stock_scanner/notify.py`
- Command routing in `app.py` and `stock_scanner/query.py`
- Scan, cache, warrant, Sheets, quota, and schedule behavior

This repo should only present, filter, preview, and deepen the existing output.

## Product Shape

The mobile UI is organized into five views:

1. Today
   - Daily selected stocks
   - START / CONT filter
   - RS, score, BOX, repeat filters
   - A/B/C entry layers, stop, pressure, risk-reward

2. Detail
   - Selected stock chart
   - Strategy facts
   - A/B/C entry layer detail
   - Original strategy reasons
   - LINE card preview

3. Warrants
   - Full warrant candidate list
   - Delta, leverage, expiry, strike, IV/HV, maker quality
   - Keeps LINE as Top 3 summary only

4. Status
   - Funnel stats
   - K-bar cache, scanner, Sheets, warrant cache, LINE quota, LIFF readiness

5. LINE
   - Shows the intended split:
     - LINE: push, commands, summary, alerts
     - Mobile Web / LIFF: detail, filtering, chart, operations

## Implementation Notes

Current implementation is still static and prototype-safe:

- `index.html` is a mobile-first single-page app.
- It loads `data/market-snapshot.js` and `data/kbar-snapshot.js` when available.
- It falls back to prototype data if snapshots are unavailable.
- It does not import or modify the fuller `justin-trading-bot` repo.

## Next Integration Step

Add a read-only export path from the fuller bot into this web app.

Suggested snapshot files:

- `data/scan-results.json`
- `data/strategy-details.json`
- `data/warrant-top3.json`

These can be generated by the fuller bot from existing Sheets/cache data without changing strategy decisions.

## Guardrails

- Do not recalculate strategy in the web app.
- Do not rewrite Flex card meaning in the web app.
- Do not invent a new LINE card format in the web app. If exact preview is needed, export the real Flex JSON from `stock_scanner/notify.py` and render from that source.
- Do not change LINE command aliases without explicit approval.
- Do not put secrets or tokens in this static repo.
- Treat mobile UI controls as read-only until LIFF auth is added.
